现在的信息系统

发表于2020-10-30|更新于2020-11-22

#

测试

什么是软件测试，目标是什么

为发现错误而执行程序的过程，根据开发各阶段的需求、设计、文档或程序内部结构，设计测试用例，利用用例运行程序以便发现错误的过程。

目标：为了发现错误而执行程序的过程；好的的测试方案能发现迄今为止尚未发现的错误，成功的测试将发现尚未发现的错误。

规范测试过程包括的活动

  1. 拟定测试计划
  2. 编制测试大纲
  3. 设计生成测试用例
  4. 实施测试
  5. 生成测试报告

信息系统测试中硬件测试、网络测试、软件测试需要做的内容

硬件测试：

  1. 配置检测
  2. 硬件设备外观检查
  3. 硬件测试，加电观察指示，执行操作

网络测试：

  1. 网络设备的外观检查
  2. 硬件测试，加电观察指示
  3. 网络连接测试，ping、ftp等

软件测试包括

  1. 单元测试（模块测试）
  2. 集成测试（联合测试）
  3. 确认测试（验收测试）
  4. 系统测试

系统测试包括

将软件与整个系统硬件、外设、支持软件、数据、人员结合起来，以需求规格说明为依据，在实际环境下进行测试，检验是否有不符合系统说明的地方。

系统测试过程：

  1. 准备
  2. 执行
  3. 返工与回归测试

内容：

  1. 恢复性测试
  2. 安全性测试
  3. 强度测试
  4. 性能测试
  5. 可靠性测试
  6. 安装测试
  7. 其它限制条件测试

平台性能评价标准，或可改善的地方

可靠性：软、硬件、数据的可靠性。
系统效率：反应完成各项功能所需的计算资源，周转时间、响应时间吞吐量等。
可维护性：确定系统错误并修正错误的难易程度。
可扩充性：处理能力和功能的可扩充程度，包括结构、硬件、软件功能的扩充性。
可移植性：从一个软、硬件移植到另一个软、硬件的难易程度。
实用性：对各部门的业务处理的提高支持程度，有效性如何。
适应性：运行环境、约束条件、需求变动的适应力。
保密性：对无意操作、硬件安全、自然灾害、外部攻击的保护措施。

系统评价

技术性能：

  1. 系统总体技术水平
  2. 系统功能覆盖范围
  3. 信息资源开发利用范围
  4. 系统质量
  5. 系统安全性系统文档资料的规范

管理效益
经济效益

系统评价的步骤

拟定评价对象，发评价通知，组成评价工作组、专家组。
拟定评价工作方案，收集基础资料。
实施评价，征求专家意见，反馈企业，撰写报告。
评价报告送咨询复核，向委托人送达报告，建立评价档案。

系统质量评价包含

  1. 对用户和业务需求的相对满意程度
  2. 开发过程是否规范
  3. 功能的先进性、有效性和完备性
  4. 性能、成本、综合效益比
  5. 运行结果有效性可行性
  6. 结果是否完整
  7. 信息资源利用率
  8. 提供信息质量
  9. 系统实用性

语句覆盖、路径覆盖

  1. 语句覆盖：设计足够的测试用例，使每条语句至少执行一次
  2. 路径覆盖：使所有可能路径至少执行一次

管理

硬件管理的基本要求

必须由专人负责管理，定期对各种设备清理、检查，确保库房、机房、设备状态正常，防止漏电、打火。对故障设备及时登记，汇报并及时处理。

所有硬件需遵循制定的管理条例。不擅自外借、挪用，非本部门人员未经许可不擅自使用部门软件硬件，严禁未授权对计算机数据、程序进行操作。

硬件设备需定期清点检测，及时处理发现问题。定期备份并做好标识。常使用设备应及时保养，各类说明书，保修卡等需统一管理立卷归档。

ISO网络管理模型（网络管理的内容）

  1. 性能管理
  2. 设备、应用配置管理
  3. 计费、利用管理
  4. 设备、应急故障管理
  5. 安全管理

网络相关设施管理包括

  1. 电源设备管理
  2. 空调设备管理
  3. 通信应急设备管理
  4. 防护设备管理

制定维护计划需考虑因素

  1. 维护背景：系统当前情况、维护对象、工作的复杂性、规模
  2. 维护工作的影响：对新系统目标、工作进度、本系统其它部分、其他系统的影响
  3. 资源需求：维护需要的时间、费用、人员

按照维护目标可将维护分为哪几类

  1. 纠错性维护:改正系统中已经发生、但测试未发现的错误。20%
  2. 适应性维护：使软件适应信息技术变化、硬件、环境管理需求的变化，而修改软件。25%
  3. 完善性维护：扩充功能、改进加工效率、改善性能而修改软甲。50%
  4. 预防性维护：提高可维护性、可靠性，适应未来软硬件变化。5%

对访问、使用计算机人员应采取的安全措施

  1. 不同用户设置不同账户
  2. 每个账户设置相关权限
  3. 启用安全审计
  4. 用户不使用或完成工作后，禁用或删除对应用户
  5. 实行监控，人员陪同、安装监控软件、重要文件隔离、备份

信息化人员的安全管理

多人负责、有限任期、职责分离、最小权限原则

  1. 加强人员审查、培训、考核
  2. 做好权限控制，签订保密合同，承诺调离后的保密任务
  3. 调离不合格人员，做好交接、回收权限、账号、钥匙、证件等材料

运行管理主要包括（内容）

  1. 出入管理
  2. 终端管理
  3. 信息管理

故障处理（管理）流程

  1. 故障处理跟踪
  2. 故障监视
  3. 故障调研
  4. 故障分析定位
  5. 故障支持和恢复
  6. 故障终止

不同监视对象有不同监视方法

监视类别 对象 方法
硬件及设备 服务器、存储设备、网络设备 通用或专用监控工具，自动监测、跟踪、报警
软件 性能指标、bug、变更需求 管理工具监测，专门测试工程师负责
人员 系统操作员、开发人员、用户、来访者、清洁运输职工 对接触过程进行跟踪记录，防止非标操作

不同类型数据库故障处理措施（数据库故障类型）

  1. 事务故障：事务在运行至正常终点前被终止，数据库处于不正常状态。内部错误、非法输入等引起。
    • a.反向扫描查找事务更新操作；
    • b.对事务操作进行逆操作；
    • c.继续查找并作相同处理；
    • d.反复处理直到查询到事务开始标记。
  2. 系统故障：也称软故障，系统停止运行的事件。硬件错误、操作系统故障、断电、DBMS代码错误。
    • a.正向扫描日志，找出故障发生前提交事务（begin transaction和commit），将事列入重做队列。将尚未完成的事务（只有begin transaction）记入撤销队列；
    • b.反向扫描，对每个undo操作进行逆操作；
    • c.正向扫描，对每个redo从新执行登记操作。
  3. 介质故障：称硬件故障。物理损坏。更换新的存储介质，利用 备份还原。

问题的控制目标

  1. 将由IT基础架构中的错误引起的故障和问题对业务的影响降低到最低
  2. 找出出故障和问题的根本原因，防止再次发生与这些错误有关的故障
  3. 实施问题预防，在故障发生前解决有关问题

问题控制步骤

  1. 发现和记录问题
  2. 归类问题
  3. 调查和分析问题
  4. 错误控制

安全审计对数据安全保障的作用

  1. 检测对系统的入侵，对潜在攻击起震慑警告作用
  2. 发现计算机滥用情况，对已发生的破坏行为提供追究证据
  3. 为系统管理提供有价值日志，协助发现入侵行为或潜在漏洞
  4. 提供系统运行统计日志，帮助管理员发现性能不足或需改进的地方

信息系统维护计划包含的内容

  1. 维护预算
  2. 维护需求
  3. 维护系统
  4. 维护承诺
  5. 维护负责人
  6. 维护执行计划
  7. 更替

系统转换

撰写原系统调研报告的目的

保证转换工作顺利实施，为制定转换计划提供必须的信息资源。

内容包括：现有的软、硬件、工作人员水平、对旧系统的熟悉程度、系统运行制度、系统数据管理等。

什么是数据迁移

对旧系统中的历史数据进行清洗、转换并装载到新系统中的过程。

转换方法：直接转换、试点后直接转换、逐步转换（分段转换）、并行转换。

新旧系统转换包含哪些内容

系统环境转换、数据迁移、业务操作转换、防范意外风险

其它

环境安全保障机制

  1. 专门用来放置计算机设备的设施或房间。
  2. 对IT资产的恰当环境保护。
  3. 有效的环境控制机制（如火灾探测）不间断电源。
  4. 定期对计算机设备的周边环境进行检查。
  5. 定时对环境保护设备进行测试。
  6. 定期接受消防管理部门的检查。
  7. 对检查中发现的问题进行处理的流程。

是否能通过访问控制避免用户误操作删除文件

不能完全避免，误操作是在用户有相应的权限下进行的操作，访问控制只能限制用户能否有控制权，限制相应的权限等。若用户不需要具备删除权限时该控制有效，若用户工作需要有删除权限时则不能实现，只能通过备份还原来恢复。

避免“信息悖论”^信息悖论的措施

通过IT财务管理对IT服务项目的规划、实施、运作进行量化管理。

IT部门角色转换，从技术部门转变为责任中心。

IT成本中心与利润中心管理的不同

IT部门组织、职责设计原则

  1. 设立清晰的远景和目标
  2. 根据服务内容思考规划部门职能，调整组织机构，清晰职责
  3. 建立目标管理、项目管理，落实目标、分解目标，建立有利的管理机制
  4. 建立人力资源管理制度、薪酬、考核制度
  5. 通过薪酬和考核体系促进绩效提高
  6. IT组织柔性化，适应IT服务需求和技术发展

Https如何确保数据安全

  1. https通过数据加密，建立一个信息安全通道，保证数据安全。
  2. 身份验证，确实网站的真实性、数据完整性，防止内容被第三方冒充篡改。从而确保安全

Https和Http访问的区别

  1. https协议使用ca申请证书。
  2. http是超文本传输协议，信息是明文传输。https具有安全性ssl加密传输协议。
  3. https协议由ssl+http协议构成比http安全。
  4. http连接简单，是无状态的。

容错途径（方法）

  1. 使用空闲设备：配置一个备用部件，平时处于空闲状态，当原部件出现错误时则取代原来部件的功能。
  2. 负载均衡：使两个部件共同承担一项任务，当其中一个出现故障时，另外一个则继续工作。
  3. 复现：两个部件执行完全相同的工作，当其中一个出现故障是，另一个继续工作。
  4. 镜像：（镜像延迟）辅助系统从原系统接受数据时存在着延时，原系统出现故障时，辅助系统就接替原系统工作，但也存在延时。
  5. 热可交换：某一部件出现故障时，可以立即拆除该部件并换上一个好的部件，使系统不瘫痪。

访问控制的类型

  1. 防御型控制：阻止不良事件发生
  2. 探测型控制：探测已经发生的不良事件
  3. 矫正性控制：矫正不良事件
  4. 管理型控制：管理系统开发、维护、使用
  5. 技术型控制：为信息技术系统、应用提供自动保护的硬件控制手段
  6. 操作控制：保护系统日常规程、机制

访问控制的层次

物理、管理、技术控制

国家禁用国外加密算法的原因

  1. 国外禁止出口密码算法和产品，所谓出口产品均有破译方法。
  2. 防范可能的国外产品、算法存在“后门”

选择基础软件产品时因该考虑的问题

  1. 软件的功能性，可扩展性，模块性，稳定性
  2. 为二次开发提供的工具，售后服务，技术支持
  3. 在以上基础上考虑价格因素，运行所需要的平台

IT人员引进依据

  2. 定期业务考核和薪酬激励，岗位业务培训
  3. IT人才规划需和企业发展相适应

企业避免泄露信息所采取的措施

  1. 严控计算机外设端口，监控审计计算机操作行为
  2. 重要信息集中存储
  3. 严格控制移动介质的使用
  4. 监控本地上网行为，监控员工上网行为
  5. 建立硬件急别的防控体系
  6. 制定合适的制度
